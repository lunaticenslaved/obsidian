## Ввод и вывод

Любая программа - это автомат, предназначенный для обработки данных. На вход они получают одну информацию, а на выход передают другую

Чтобы на вывод не передалась управляющая последовательность (особенно при чтении нетекстовых файлов) лучше воспользоваться командами `hexdump` (представляет вывод в виде последовательности шестнадцатеричных ASCII кодов) и `strings`, которая показывает только те части файла, которые могут быть представлены в виде текста.

## Перенаправление ввода и вывода

Все данные вводятся и выводятся из процесса через открытые потоки данных - дескрипторы.

При открытии обычного файла или файла-дырки создаются дескрипторы и дальнейший ввод/вывод идёт через них. Когда файл закрывается, дескриптор закрывается и может быть использован снова для другого файла.

Всего существует три вида дескрипторов:

- стандартный ввод `stdin`,
- стандартный вывод `stdout`,
- стандартный вывод ошибок `stder`.

При копировании окружения для запуска дочерних процессов идёт также копирование всех открытых дескрипторов от родительского процесса дочернему.

Так как стартовый командный интерпретатор является файлом-дыркой, то любая команда, запущенная из него, будет получать данные с ввода терминала, а любая команда, запущенная интерактивно (не в фоне) использовать вывод терминала.

#### Стандартный вывод

**Стандартный вывод** - это поток данных, открываемый системой для каждого процесса в момент его запуска, и предназначенный для данных, выводимых процессом.

Стандартный вывод может быть перенаправлен. Для этого используется команда `>` или `1>`.

#### Стандартный ввод

**Стандартный ввод** - это поток данных, открываемых системой для каждого процесса в момент его запуска, и предназначенный для передачи данных процессу.

#### Стандартный вывод ошибок

Стандартный вывод ошибок - это поток данных, открываемый системой для каждого процесса в момент его запуска, и предназначенный для диагностических сообщений, выводимых процессом: ошибок, предупреждений и т.д.

Стандартный вывод ошибок может быть перенаправлен с помощью команды `2>`.

Объединить стандартный вывод и стандартный вывод ошибок можно с помощью `2>&1` или `&>` или `>&` в зависимости от терминала.


## Перенаправление в никуда

Если не нужно выводить в терминал какие-то данные, то можно направить их в специальный файл-дырку, предназначенный для удаления ненужных данных `> /dev/null`.
